## 本番進行フロー

### 前提
vibec.uk はロゴ + タイトル + Discord CTAだけのスカスカ状態からスタート。

### タイムライン

#### 1. 現状を見せる（1分）
- vibec.uk を映して「これが今のサイトです。スカスカですよね」

#### 2. デザインの方向性を観客に聞く（2-3分）
- 「どんな雰囲気にしたい？」→ 選択肢を見せる
- **テーマ候補:** サイバーパンク / 和風 / レトロ / ミニマル / ポップ / ブルータリズム / ターミナル / 宇宙
- カラー・フォントも口頭で拾う

#### 3. デザイン変更プロンプトを投げる（3-5分）
- 観客が選んだキーワードでプロンプトを組み立てて実行
- サイトがガラッと変わるのをホットリロードで見せる

#### 4. コンテンツを聞く（2-3分）
- 「次は何を足したい？」→ イベント情報 / About / メンバー紹介 / FAQ など
- 観客から自由な意見も拾う

#### 5. セクション追加プロンプトを投げる（残り時間）
- 観客のアイデアをプロンプトにして追加
- 時間の許す限り繰り返す

### 技術メモ
- 開発サーバー: `npm run dev` → localhost:4321
- 元に戻す: `git checkout src/pages/index.astro src/styles/global.css`
- 本番デプロイ: `main` にプッシュ → GitHub Actions → vibec.uk に自動反映
